## Project Overview

A hands-on data engineering tutorial series using GCP, implemented in Python. The series processes UCI Online Retail Dataset (UK retail transactions) through a 3-part pipeline:

- **Part 1 (this repo's current scope)**: Ingest raw data into GCS and profile data quality
- **Part 2**: Cleanse data with Cloud Run Functions and load into BigQuery
- **Part 3**: Build aggregation marts with BigQuery SQL and visualize in Looker Studio

The code is also published as a Zenn article (`gcp-de-intro-01-gcs-profiling.md`) targeting readers who are new to GCP but comfortable with pandas.

## Setup

```bash
uv sync
gcloud auth login
gcloud config set project YOUR_PROJECT_ID
```

Download `Online_Retail.xlsx` from the UCI repository (see `data/README.md`) and place it at `data/Online_Retail.xlsx`.

## Commands

```bash
# Split xlsx into daily CSVs and generate master files (Part 1)
uv run python 01_profiling/scripts/split_data.py

# Upload to GCS (requires BUCKET_NAME env var) (Part 1)
export BUCKET_NAME="de-handson-gc-landing"
uv run python 01_profiling/scripts/upload_to_gcs.py

# Run profiling notebook (Part 1)
uv run jupyter notebook 01_profiling/notebooks/profiling.ipynb

# Deploy Cloud Run Function (Part 2)
gcloud functions deploy clean-and-load \
  --gen2 --runtime=python312 --region=asia-northeast1 \
  --source=02_cleaning/cloud_run_function --entry-point=main \
  --trigger-http --allow-unauthenticated --memory=512M \
  --set-env-vars PROJECT_ID=${GOOGLE_CLOUD_PROJECT},BUCKET_NAME=de-handson-gc-landing,DATASET_ID=de_handson

# Batch load all dates to BigQuery (Part 2)
export PROJECT_ID="${GOOGLE_CLOUD_PROJECT}"
bash 02_cleaning/scripts/batch_clean_and_load.sh

# Lint
uv run ruff check 01_profiling/ 02_cleaning/
uv run ruff format 01_profiling/ 02_cleaning/
```

## Architecture

GCS is a **landing zone (Data Lake)** for raw source files only. Transformation and aggregation happen in BigQuery (DWH).

```
GCS: gs://de-handson-gc-landing/        BigQuery: data-engineering-handson-gc
└── raw/  ← Part 1: immutable, never overwritten    ├── processed dataset  ← Part 2
    ├── transactions/transactions_YYYY-MM-DD.csv     └── mart dataset       ← Part 3
    ├── products.csv
    └── countries.csv
```

Local data directory mirrors the GCS raw layer:

```
data/
├── Online_Retail.xlsx    ← not committed, downloaded by user
├── transactions/         ← generated by split_data.py
├── products.csv          ← generated by split_data.py
└── countries.csv         ← generated by split_data.py
```

Part 1 under `01_profiling/`:
- `scripts/split_data.py` — reads xlsx, filters to 2010-12, writes daily CSVs + master files
- `scripts/upload_to_gcs.py` — uploads local data to `raw/` prefix; requires `BUCKET_NAME` env var
- `notebooks/profiling.ipynb` — loads all daily CSVs locally, profiles missing values / anomalies / date distribution

Part 2 under `02_cleaning/`:
- `scripts/batch_clean_and_load.sh` — calls Cloud Run Function for each daily CSV; requires `PROJECT_ID` env var
- `sql/schema.sql` — BigQuery `de_handson.transactions` テーブル定義
- `cloud_run_function/main.py` — reads CSV from GCS, cleanses, loads to BigQuery
- `cloud_run_function/requirements.txt` — generated by `uv export --no-hashes --no-dev`

## Code Conventions

- **No docstrings** — type hints carry the "what", article prose carries the "why"
- **Comments only for WHY** — not what the code does
- ruff line length: 88, target Python 3.12
- ruff lint selectors: E, F, I (errors, pyflakes, isort)
- All scripts use `pathlib.Path`, not string paths
- Quality check before marking scripts done: `uv run ruff check scripts/` must be warning-free

## Data Notes

- Dataset covers 2010-12-01 to 2010-12-23 (20 days; Saturdays have no data, Sundays do)
- `CustomerID` is ~37% null — treat as guest purchases, not corrupt data
- Negative `Quantity` rows (798) correspond to `InvoiceNo` starting with `C` (cancellations, 728 rows)
- `UnitPrice = 0` rows (273) exist and require a business rule decision in Part 2
